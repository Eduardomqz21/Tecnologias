¿Qué es SweetAlert2?

Es una librería para mostrar alertas modernas dentro de la página. Se usa cuando quieres pedir confirmación, mostrar éxito o error, capturar texto o enseñar un mensaje breve sin depender de los diálogos nativos del navegador.

La idea es simple: el usuario hace una acción y recibe una respuesta clara, bonita y fácil de entender. Eso mejora mucho la experiencia frente a un alert() tradicional.

Resumen rápido: cuándo usar cada tipo

Mensaje simple

Cuando solo quieres avisar algo breve o saludar al usuario.

Confirmación

Cuando una acción puede borrar, cancelar o cambiar algo importante.

Entrada de datos

Cuando necesitas un nombre, correo, opción o comentario.

Toast / carga

Cuando el mensaje debe ser ligero o aparecer mientras ocurre algo más.

Mensajes simples

Úsalo para saludar, avisar algo simple o dar un aviso corto y visible.

Swal.fire('SweetAlert2 está funcionando');

Estados e iconos

Sirven para decir rápido si algo fue correcto, informativo, de advertencia o error.

Swal.fire({
  icon: 'success',
  title: 'Listo',
  text: 'La operación terminó bien'
});

Decisiones importantes

Se usa antes de eliminar, cancelar o hacer cambios que no se pueden deshacer.

Swal.fire({
  title: '¿Seguro?',
  text: 'No podrás revertir esto',
  icon: 'warning',
  showCancelButton: true,
  confirmButtonText: 'Sí, eliminar'
}).then((result) => {
  if (result.isConfirmed) Swal.fire('Eliminado', '', 'success');
});

Formularios y entradas

Ideal para pedir nombre, correo, una elección o un comentario corto.

const { value } = await Swal.fire({
  title: 'Escribe tu nombre',
  input: 'text',
  inputPlaceholder: 'Tu nombre'
});

Notificaciones pequeñas

Úsalo para avisos pequeños, no bloqueantes, normalmente en una esquina.

Swal.mixin({
  toast: true,
  position: 'top-end',
  showConfirmButton: false,
  timer: 2500
}).fire({
  icon: 'success',
  title: 'Guardado'
});

Timer y carga

Sirve para procesos automáticos o para simular carga mientras se espera una respuesta.

Swal.fire({
  title: 'Cerrando en 2 segundos',
  timer: 2000,
  timerProgressBar: true
});

Patrones reutilizables

Un mixin sirve para no repetir configuración. Si vas a usar siempre el mismo estilo para toasts, confirmaciones o botones, lo defines una vez y lo reutilizas. 

const toast = Swal.mixin({
  toast: true,
  position: 'top-end',
  showConfirmButton: false,
  timer: 3000
});

toast.fire({ icon: 'success', title: 'Guardado' });

HTML personalizado

Cuando necesitas más control, puedes usar HTML dentro de la alerta. Esto sirve para poner enlaces, listas, texto con formato o contenido más largo. 

Swal.fire({
  title: '<strong>HTML</strong>',
  html: 'Puedes usar <b>negritas</b> y <a href="#">enlaces</a>'
});

Resultado y cierre

SweetAlert2 no solo muestra ventanas bonitas. También devuelve información útil sobre lo que hizo el usuario: si confirmó, canceló, cerró o dejó que el tiempo terminara. 

Swal.fire({...}).then((result) => {
  if (result.isConfirmed) {
    // acción aceptada
  }
  if (result.isDismissed) {
    // el usuario cerró o canceló
  }
});
